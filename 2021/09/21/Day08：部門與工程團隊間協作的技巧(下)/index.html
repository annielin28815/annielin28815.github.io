Day08：部門與工程團隊間協作的技巧(下)

Date: 2023-11-21  19:46:08 Category: 新手全端工程師的職場成長歷程

一、前言

承上一篇的部門與工程團隊間協作的技巧(上)已有稍微提到一些工程師間的協作軟體工具，那如果是和行政、企劃或專案管理部門的同事，大家有用過什麼工具呢？

二、部門之間的協作軟體或工具

(一) Trello

  • 有免費版、付費版的功能差異。我只用過免費版。
  • 支援 markdown 語法，可以張貼整齊的程式碼工團隊間討論。
  • 卡片功能容易清楚分辨任務排序與分類。
  • 卡片及內容都可以隨需求拖拉，很容易上手。
Trello的網頁版

(二) Asana

  • 鮮明的白底紅logo配色，但似乎目前僅支援google第三方登入。
  • 相同的有提供看板(任務)功能，可以隨時與團隊的其他人共組一項專案任務，以便追蹤進度。
  • 亦有免費版、付費版的功能差異。我僅用過付費版大約一個多月，個人覺得功能差異甚遠，可以參考官方網站說明斟酌是否付費。
  • 有提供mac桌機版，但我尚未使用過。
  • 功能相較 Trello 來得齊全，有主管審核制度等。
Asana的網頁版

(三) Jira

  • 亦有免費版、付費版的功能差異。
  • 聽說近期較為廣泛被使用之任務管理工具，但我個人覺得沒有付費，功能實在超不實用！

(四) Figma 和 Adobe XD

  • 兩者功能類似，但也各自擁有一派支持者，主要用來設計網站 UI 介面。
  • 如果我是前端工程師，我其實比較想收到使用 Adobe XD 畫出來之設計稿，因為有滿多程式碼與快捷鍵方便使用。
  • 如果我是短暫被分派到設計網站 UI 介面，我個人恩會選則使用 Figma ，主要是因為免費版與開源資源非常多！

(五) Chrome 遠端桌面

  • 主要是用在會議中與同事分享單一畫面或討論某個網頁時，我個人覺得很好用。例如畚箕開發的程式碼，不必為了某次討論一直打包上傳更新到AWS等。
  • 使用方法詳點大標即可看到。

(六) Gitmind 和 Xmind

  • 兩種都是畫心智圖很好用的工具，我覺得用在團隊討論上也是超有幫助！
  • 兩者都有提供軟體版或網頁版，個人覺得 Gitmind 功能與資源都較多一點。
  • 因為手邊電腦沒有示意圖，後續將找時間再行補上。

三、結論與自我鼓勵

今天是2021年中秋連假最後一天，回到台北後仍然拿起電腦，像過去幾天一樣，分享我當一個新手全端工程師所學到的一些微小事物，關於團隊間的合作與溝通平台，就在這兩篇簡短的介紹完了。那大家如果有推薦的其他協作工具，也可以在下面留言分享唷！
