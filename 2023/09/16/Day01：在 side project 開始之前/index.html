Day01：在 side project 開始之前

Date: 2023-09-16  23:36:41 Category: 用 React 和 Firebase 打造一個完整 side project 吧！

30天可以做什麼？

身為一位工程師，最需要的就是以 side project 來展示自己的技能，在一年一度的鐵人賽中看見 sideProject30 這個新主題，當然就興奮的報名啦！

以往開發都有同學、同事一起，要說靠自己完成一個完整專案的經驗，可說是沒有？工作上，總是有專案經理、UI/UX 設計師、前端與後端工程師等多位專業夥伴，一起規劃網站架構並分工開發。常常因為工作分配，無法理解網站實作上的其他完整流程，所以這次就決定試試看從網站的構思到開發，都由自己一手全包吧！

因為自己很常有訂位的需求，所以想做一個自己都覺得好用的訂位網站，雖然開發時間較短，也許有些功能和細節沒辦法在第一時間處理的很好，但希望透過30天的過程，我可以將有興趣的作品初步實作完成並與大家分享。

主題發想

現今生活中，有很多髮廊、美業、手作體驗或甜點課程等，皆有提供預約訂位的服務，常見都是使用 Line 或其他社群平台的私訊功能。然而無論是電話預約或文字訊息，皆是建立在雙方信任間進行。在於店家(業主)角色一定會擔心為顧客保留時間或預留材料、顧客卻臨時因為一些特定狀況沒有出席，那提前準備的一切將會直接轉為損失；換做顧客的角度，一定也會不想單純為了久久一次的訂位需求，使用完全不同的網站或 app、申請個人帳號等繁瑣動作。

基於上述情境，雙方皆有擔憂的狀況下，我發想了這個 side project 的主題，希望可以做到以下主打功能：

  • 顧客：針對有興趣的項目，不需再私訊等待品牌業者回覆，可直接於網站上直接找到還可以預約的日期時段。
  • 店家：可維持原有的自媒體經營模式、官方帳號與呈現作品集的方式，僅將「預約」功能獨立出來。
  • 預約流程：店家可定義預約服務的訂金，顧客預約後須於3日內回填匯款資訊，並經店家查核確認，才算完成預約，用以降低爽約機率。
  • 身分驗證：為預約流程添加電話號碼驗證功能，盡可能降低爽約可能。

競品分析

完整的作品須經過市場上現有的競爭品功能差異比較、分析，但是因為時間有限，本次專案不會著墨篇幅於這部分。就以預約平台來說，個人超喜歡現在很多餐廳在 google map 或官網上所提供的 inline app 連結，該網站功能很豐富完整、使用者介面卻很簡單明瞭，它可以解決有些店家忙碌無法接聽預約電話的問題；另外也方便顧客在不方便通話時多一個預約的方式選擇(像是會議結束前，想先預訂餐廳的情形等)。

因為本次專案比較想主打一些有簡易工作室、或使用租借會議空間提供服務的店家與業者這種客群，這些店家普遍都有經營自己的自媒體、官方網站，他們所提供的服務項目可能原本是使用簡易的 google 表單，或者是目前臺灣較常見的 line 官方帳號，但就如同開頭提到的總總問題，我希望這個即將開始進行的網站，可以讓店家們多一個更有系統的預約平台選擇。

開賽期許

值得一提的是，我並不會以致敬哪個現有網站為目標、作出和他們一樣強大又全面的功能，而是單純將預約系統獨立出來，希望站在使用者的角度，讓店家和顧客都可以透過這個網站輕鬆快速的預約。
